Questionnaire

Questionnaire pages

A questionnaire page will have the following items:

  • Breadcrumbs
  • Page title
  • Lead text
  • Questions, see below (one or multiple)
  • Actions
    • Primary (ex: Save and continue)
    • Secondary (ex: Cancel, Go back)
    • Tertiary (ex: Skip)

Questions

Pertains to components and subcomponents that make up the questionnaire feature. Questionnaire components include the following:

  • Question

    The question is the primary ask of the user. There must be a direct relationship between the question and the response solicited from the user. Questions should be focused and direct, and should not include text such as "for example." If additional context is needed, please use the question support text.

  • Quesiton support text

    Used for additional description or supporting information, such as definitions, example lists, etc., or any important information that the user needs know before providing a response. Quesiton support text will include HTML support for:

    • <p>
    • <ul>
    • <ol>
    • Links, bold text, italicized text
  • Response

    The response is where the user provides a response to the question being asked. The response options must relate directly to the question being asked. The response will support a broad range of inputs, including:

    • Radio buttons
    • Check boxes
    • Text input
      • Text
      • Text area
      • Date
      • Date Range
      • Certify components:
        • Document uploader
        • Table
        • Percentage text input (coming soon)
        • USD text input (coming soon)
  • Question follow-up

    Question follow-ups remain hidden and disabled on the page and only appear when the user makes a specific response that triggers it to appear. The follow-up may be text or another question or set of questions. Question follow-ups use the same model as primary questions.

  • Footnotes

    Think of these as a footer section for a question. It includes links to the CFR and other information that might be useful to the user if the question support text do note convey yada yada.
